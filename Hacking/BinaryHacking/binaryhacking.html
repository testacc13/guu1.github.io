Defeating Non-Executable Stack

Known as Return to libc - It will make use of libc library to export our syscalls. Instead of returning control to instructions on the stack, as a traditional stack buffer overflow, we will force the program to return to an address that corresponds to a specific dynamic library function. That dynamic library function will not be on the stack, circumventing any stack exection restrictions.
